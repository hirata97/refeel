# Issue #185: [親チケット][Feature] 感情タグ機能の完全統合

## 概要
## 📝 機能概要
感情タグ機能は日記作成時の選択のみ実装済みですが、日記閲覧・編集・フィルタリングでタグが活用できていません。本チケットでは感情タグ機能を完全統合し、ユーザーが一貫して感情タグを活用できる機能を実装します。

## 🎯 目的・背景
- 現在、感情タグは作成時に選択可能だが、閲覧・編集時に表示・編集できない
- フィルタ機能でも感情タグでの絞り込みができない
- ユーザビリティの大幅向上と感情分析機能の価値最大化が必要

## ✅ 受け入れ条件
- [ ] 日記一覧で感情タグがチップ表示される
- [ ] 日記詳細モーダルで感情タグが表示される
- [ ] 日記編集時に既存の感情タグが読み込まれ、編集可能
- [ ] フィルタ機能で感情タグによる絞り込みが可能
- [ ] モバイル対応（レスポンシブデザイン）
- [ ] アクセシビリティ対応

## 🎨 UI/UX要件
- 感情タグは色分け表示（positive=green、negative=red、neutral=blue）
- 一覧表示では3つまで表示、それ以上は「+N個」で省略
- 詳細画面では全て表示
- タグクリックで該当タグでフィルタリング（将来機能として予約）

## 🔧 技術的要件
- 既存のemotionTagsStore活用
- パフォーマンス最適化（JOINクエリ使用）
- TypeScript厳格モード対応
- 既存コンポーネントとの一貫性維持

## 🧪 テスト要件
- [ ] 新規コンポーネントのユニットテスト
- [ ] 統合機能のE2Eテスト
- [ ] レスポンシブデザインテスト

## 📚 参考資料
- EmotionTagSelectorコンポーネント（既存実装）
- src/stores/emotionTags.ts（既存ストア）
- 感情タグテーブル設計（database/）

## 🚀 実装フェーズ
1. Phase 1: EmotionTagChipsコンポーネント作成（基盤）
2. Phase 2: 日記閲覧機能での表示
3. Phase 3: 日記編集機能の統合
4. Phase 4: フィルタ機能拡張
