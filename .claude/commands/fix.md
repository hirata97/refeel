# バグ修正・エラー解決自動化

現在発生している問題やエラーを自動的に検出・分析・修正します。

## 問題検出フロー

### 1. 自動診断
```bash
npm run ci:all          # 全品質チェック実行
git status             # 未コミット変更確認
npm run test:unit      # テスト実行状態確認
```

### 2. エラー分析
- TypeScriptコンパイルエラー
- ESLintルール違反
- テスト失敗箇所
- ビルドエラー原因
- 実行時エラーログ

### 3. 根本原因分析
- 依存関係の問題
- 型定義の不整合
- コンポーネント間の結合問題
- Supabase接続・認証問題
- 環境設定の不備

## 修正実行フロー

### TypeScriptエラー修正
- 型定義の追加・修正
- インターフェース設計の見直し
- 自動生成型の適用

### Vue.jsエラー修正
- コンポーネントのリアクティビティ問題
- Piniaストア状態管理の修正
- ルーティング問題の解決

### テストエラー修正
- モックの設定修正
- テストデータの調整
- アサーション条件の見直し

### ビルドエラー修正
- 依存関係の解決
- 設定ファイルの調整
- 型定義ファイルの更新

## 品質保証

### 修正後検証
```bash
npm run generate-types  # 型定義再生成
npm run ci:lint        # リント確認
npm run ci:type-check  # 型チェック
npm run test:unit      # テスト実行
npm run build         # ビルド確認
```

### 回帰テスト
- 既存機能への影響確認
- 関連コンポーネントのテスト
- E2Eテストの実行

## Root Cause Analysis

修正完了後、以下を記録：
- **問題の根本原因**
- **修正内容の詳細**
- **今後の予防策**
- **学習した教訓**

現在のプロジェクト状態を診断して、発見された問題を自動的に修正開始してください。
